# 🚀 Day 17 - Environment Based Workflow

## 📌 Overview
This workflow demonstrates how to handle different environments (development and production) based on branch conditions.

## 📂 Directory Structure
```bash
30-Days-of-GitHub-Actions/
├── .github/
│   └── workflows/
│       └── day-17.yml
└── Day-17/
    └── README.md
```

## ⚙️ What it does
- Triggers on push to `main` and `dev`
- Detects branch name
- Executes environment-specific steps
- Simulates deployment process

## 📂 Files
- `.github/workflows/day-17.yml` → Workflow definition  
- `Day-17/README.md` → Documentation  

## ▶️ Key Steps
1. Identify branch  
2. Apply conditional deployment  
3. Run environment-specific steps  
4. Complete workflow execution  

## 🎯 Purpose
To understand:
- Environment-based pipelines  
- Branch-based deployment  
- Conditional execution in CI/CD
