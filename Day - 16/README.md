# 🚀 Day 16 - Docker Tag and Push Workflow

## 📌 Overview
This workflow demonstrates how to tag Docker images and simulate pushing them to a container registry.

## 📂 Directory Structure
```bash
30-Days-of-GitHub-Actions/
├── .github/
│   └── workflows/
│       └── day-16.yml
└── Day-16/
    └── README.md
```

## ⚙️ What it does
- Triggers on push to `main`
- Builds a Docker image
- Tags the image with a version
- Simulates pushing to a registry

## 📂 Files
- `.github/workflows/day-16.yml` → Workflow definition  
- `Day-16/README.md` → Documentation  

## ▶️ Key Steps
1. Build Docker image  
2. Tag image with version  
3. Simulate push to registry  
4. Verify images  

## 🎯 Purpose
To understand:
- Docker image tagging  
- Versioning in CI/CD  
- Registry push workflow concept
