# 🚀 Day 24 - Permissions Workflow

## 📌 Overview
This workflow demonstrates how to control permissions in GitHub Actions for improved security.

## 📂 Directory Structure
```bash
30-Days-of-GitHub-Actions/
├── .github/
│   └── workflows/
│       └── day-24.yml
└── Day-24/
    └── README.md
```

## ⚙️ What it does
- Triggers on push to `main`
- Defines restricted permissions
- Limits access to repository resources
- Simulates secure operations

## 📂 Files
- `.github/workflows/day-24.yml` → Workflow definition  
- `Day-24/README.md` → Documentation  

## ▶️ Key Steps
1. Define permissions at workflow level  
2. Restrict access to required scopes  
3. Execute workflow with limited privileges  

## 🎯 Purpose
To understand:
- Permissions in GitHub Actions  
- Security best practices  
- Least privilege principle
